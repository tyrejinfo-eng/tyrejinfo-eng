# Security Policy

## Supported Versions

Security updates are applied to the latest active development branches of all TyRejinfo Engineering repositories.

Older versions may not receive security patches.

---

# 🛡 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Do NOT open a public issue.

Instead, report security concerns directly to:

📧 **[ty.rejinfo@gmail.com](mailto:ty.rejinfo@gmail.com)**

---

# 📌 What to Include

When reporting a vulnerability, please include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Affected repository/module
* Possible mitigation suggestions (if available)

---

# 🔍 What Happens Next

Once reported:

1. The issue will be reviewed privately
2. Severity will be assessed
3. A fix will be developed if required
4. A patch will be released
5. Credit may be given if appropriate (unless anonymity is requested)

---

# ⚠️ Scope

Security policy applies to:

* MonsterCoder
* MonsterTools
* AIBlue Copilot
* SGL-DevEngine
* Supporting libraries and tooling

It does NOT cover:

* Third-party dependencies
* External services
* User-created plugins (unless explicitly integrated)

---

# 🔒 Responsible Disclosure

We ask that researchers:

* Do not publicly disclose vulnerabilities before a fix is available
* Avoid disrupting running systems or repositories
* Respect user data and privacy

---

Thank you for helping keep the ecosystem secure.
